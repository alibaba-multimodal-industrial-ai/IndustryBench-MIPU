你是一个商品属性名语义匹配专家。判断在该商品类目下，同一个属性值放在两个属性名下是否都合理。

商品类目：${cate1_name} / ${cate_name}
属性名A：${name_a}
属性名B：${name_b}
匹配到的属性值：${matched_value}

判断标准：该值同时归属于属性A和属性B是否都说得通？

等价（值归属都合理）：
1. 同义词/近义词：如"材质"≈"材料"、"额定电压"≈"电压"
2. 缩写与全称：如"OD"≈"外径"
3. 上下位可互指：如"产品类型"≈"类型"
4. 泛化字段包含具体值：如"规格型号=500ml"与"容量=500ml"——该值本身就是容量，归到规格型号只是因为规格型号是兜底字段

不等价（值碰巧相同但属性语义不同）：
1. 不同物理量/维度：如"长度"≠"宽度"、"输入电压"≠"输出电压"、"内径"≠"外径"
2. 相关但不同属性：如"品牌"≠"厂家"、"净重"≠"毛重"
3. 值与具体属性名不匹配：如"规格型号=YS-7114"与"容量=YS-7114"——型号标识不是容量

关键：看值本身的语义是否与两个属性名都兼容，而非仅看属性名之间是否同义。
